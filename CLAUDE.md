## 1. 产品边界(BOUNDARY · 不可漂移)

**做什么(本期内)**:

- 检索质量 + 评估体系。**评估优先**:先尺子、先诊断,再优化。
- 多模态 = **多格式归一**(文档抽取 / 扫描件 OCR / 照片 EXIF / 视频文件名·日期 / 媒体仅元数据·纯噪声恒真负例)进一个可检索索引。
- 证明核心命题:**真实找文件大量非语义 → 混合检索 > 纯语义**。

**不做什么(本期外 / 已否定 / 仅适配)**:

- ❌ 上线产品、远程传输、权限 / 多租户、产品化、部署。
- ❌ CLIP 视觉语义检索、视频内容(ASR/抽帧)与拍摄地点、音频转写 —— 一律可选 / 二期,**不得擅自提前实现**。
- ❌ 速度 / 端侧优化(二期;本期仅"分段计时"测量,不做优化)。
- ❌ 重型图文跨模态融合 —— 不做,也不在叙述里过度宣称。
- ❌ 回答生成质量(主线不做)—— 仅收尾**可选**小验证,不影响 DoD(连主线:更好检索 = 抑制幻觉)。

**已否定方向(禁止把项目带回)**:做成"又一个无评估的 demo";注水 / 又圆又大的裸数字;按"存储占比"冒充"可检索能力"。

需要触碰边界时,先停下问用户(见 §9),不得擅自实现。

## 3. 项目特有铁律(本项目的灵魂 · 高于一般建议)

违反即破坏项目价值,按【阻断问题】处理:

1. **评估优先且因果有序**:尺子(评测集 + GT)先于系统;**GT 在跑检索前锁死、绝不回改以迁就结果**。**dev/test 分割不可逾越:调参与诊断只在 dev,test 集只在最终报告一次,严禁在 test 上反复调参刷数字**。先优化后补评估 = 注水。
2. **诚实数字**:任何指标必附 baseline + 测试集规模 + **仍失败的案例**;杜绝"又圆又大"的裸数字(如 62%→94% 无方法说明)。
3. **存储占比 ≠ 可检索角色**:语料按存储分类,评估按"可检索角色"分类。**「文件类型 × 查询类别」资格矩阵以 PRD §6.1 + ARCHITECTURE AD-5 为权威、与 GT 一起逐格锁死**。视频**退出"所需分量含地点 / 语义"的查询正例**(Q4 / Q5,及任何 `required_components` 含 location/semantic 的 Q6);**仅当某 Q6 所需分量 ⊆ {time, type, filename} 时视频才进该 Q6 正例**;媒体**全类恒真负例**。真负例**只压测 precision,绝不进 recall 正例**;**严禁给真负例贴合成信号充正例**(如给视频造 GPS)。机制细节(分量级 `supported_components`、派生期筛正例、评估期成员判定)**以 ARCHITECTURE AD-5 为准**;一致性判据:PRD §6.1 中类型 T 对 Qx 标 ✅ ⟺ `required_components(Qx) ⊆ supported(T)`,S2 冻结前逐格核对,任一格不满足即冲突、停下对账。
4. **仿真数据如实披露**:语料为公开数据 + 合成;合成字段(日期 / 文件名 / 票据内容等)必须在 manifest 标注来源;评测查询为半合成须说明。
5. **窄而深 > 广而浅**:CLIP / 视频内容 / faithfulness 一律可选、置后;**在检索 + 评估核心做深前,禁止擅自 broaden**。
6. **多模态老实定位**:多格式归一,非重型视觉-语言融合;对内对外都不过度宣称。
7. **借现成 vs 自己做**:管线骨架 / 向量机制 / embedding / 数据下载 = 借现成;清洗 / 分块 / 检索策略 / 评测集设计 / 诊断 / 优化 = **灵魂,由用户拍板、关键代码用户过目**;不得替用户做灵魂决策。
8. **语义 GT(QA→文件)映射如实降级**:把 SQuAD / DuReader / DocVQA 的段落 · 版面**物化为独立文件(1 问题 = 1 文件)**做 file-level GT,机制是**半自动映射 + 抽检手标**;**严禁宣称"全自动 / 免手标"**。S2 前先跑 10 条映射探针验证可行性,不通过即停下问用户(ARCHITECTURE AD-1/§11、PRD §6.1)。
9. **基线隔离**:`SemanticRetriever` **只投影 `text`、严禁读 `structured`**,并以契约测试(`structured` 置空 / 打乱断言基线结果不变)锁死。基线偷读结构化条件 = "纯语义 vs 混合" A→B 失真,按**阻断**处理(ARCHITECTURE AD-3)。
10. **真负例必须进索引**:媒体 / 视频(真负例类)与抽取失败的坏文件(带 `extract_status`,明细入 `errors.jsonl`)**照常进索引**,与可正例文件无差别参与召回,仅在 GT 派生与归因处被矩阵判真负例。排除它们 = 掏空 precision 压测基础,按**阻断**处理(ARCHITECTURE §5 不变量③)。

## 7. 设计 / 实现规则(继承 + 适配)

- 从合同 / schema / 核心模型起步,不从脚本 / notebook 倒推核心。
- 共享语义单一真源;adapter / CLI / notebook 只做接线与展示,不拥有业务语义。
- **生成物 / 下载数据只读不手改**;改源重生成。
- 一切外部输入 / 下载 / 缓存有上界与降级;**数据本地优先(数据不出门)**。
- 生产路径用结构化错误(stage / context);禁止裸 `except` / `unwrap` 式断言 / 静默吞错处理可恢复问题。
- 配置 / 密钥显式下传或走配置系统;禁止硬编码密钥、全局可变静态。
- 指标 / 诊断报告必须来自**真实运行数据**;拿不到给 `N/A` / 空集 / 结构化不可用原因,禁止占位 / 伪健康值。

> 适配块采用精简格式:统一 `authority` = 低于本宪法、高于普通建议;`verification` 统一见 §10 验收与 conda 块,各块不再重复;且当且仅当项目真实使用该技术时启用。

## 8. 错误分级(审查 / 验收用)

- **设计风险**:导致架构漂移 / 范围蔓延(擅自 broaden)→ 说明取舍与验收入口。
- **可记录债务**:不影响本轮目标链路 → 说明不处理原因与后续入口。
- **无关优化**:不进本轮,禁止借机扩大。

不用情绪强度替代工程优先级:无证据的担忧不写成阻断;有证据的阻断不降级。
