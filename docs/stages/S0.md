**执行目标**:`conda env create` 建 `nas_rag`(3.12);建目录骨架 + `config` 常量;`import nas_rag` 成功、空 `pytest` 绿。

**步骤**
1. `conda env create -f environment.yml` 建 `nas_rag`(3.12),不污染 base。
2. 目录骨架(空 `__init__.py`):`src/nas_rag/{domain,corpus,ingest,index/vector_store,retrieval,eval,timing,pipelines}`、`tests/`(镜像 + `fixtures/`)、`config/`、`data/{raw,corpus,index}`(gitignore 占位)、`eval/frozen/`、`docs/stages/`。
3. `config/settings.py` 常量(单一真源):路径、`EMBED_MODEL`/`RERANK_MODEL`、`K_VALUES=[1,3,5,10]`、`PRIMARY_K=5`、`fusion={method:"rrf", k_rrf:60}`、`rerank_candidate_n`、`MIN_CANDIDATE_POOL`、`{MIN_PER_CATEGORY:8, MIN_TEST_PER_CATEGORY:2, TEST_RATIO:0.2}`、分词配置、语料配方比例占位;`config/type_labels.py`(受控词表枚举占位)、`config/capability_matrix.yaml`(分量级 `supported_components`,按 ARCHITECTURE AD-5 填)。
4. `pyproject.toml`(`pytest` `pythonpath=["src"]`、`testpaths=["tests"]`)、`.gitignore`(`data/raw|corpus|index`、模型权重、`__pycache__`)、`README.md` 骨架。

**任务边界**:只建空骨架 + 配置常量;**不写业务逻辑、不下载数据、不建索引**。
**验收**:`conda env create` 成功;`python -c "import nas_rag"` 成功;`python -m pytest`(占位/收集)绿;目录结构齐;`config` 常量就位、无散落硬编码。

## 子阶段 S0.2 — 装包冒烟门禁 + OCR 引擎选定

**执行目标**:确认全部核心依赖(含 OCR 回退链、reranker CrossEncoder)**装得上且 import 得了**;选定可用 OCR 引擎。

**步骤**
1. 核心依赖装包验证:llama-index 原语(core/readers-file/retrievers-bm25)、chromadb、sentence-transformers、rank-bm25、**jieba**、datasets、huggingface-hub、pypdf、pillow、pandas、numpy。
2. **OCR 回退链**:按 `PaddleOCR → RapidOCR → Tesseract+chi_sim` 顺序试装,选第一个"装得上 + import 成功"的,记入 `config.OCR_ENGINE`。
3. **reranker**:确认 sentence-transformers `CrossEncoder` 能定位 `bge-reranker-v2-m3`(import + 实例化路径,**不引 FlagEmbedding**);embedder 确认 `bge-m3` **dense** 加载路径。
4. `tests/test_smoke_imports.py`:纯导入冒烟 + 关键对象轻量实例化。

**任务边界**:只验证"装得上 + import + 轻量实例化",**不跑真实推理 / 不强制下大权重**(能 lazy 就 lazy);OCR 只选引擎、不处理真实图。顺带核对 `llama-index-vector-stores-chroma` 是否冗余(按 AD-2 `ChromaStore` 走原生 chromadb,倾向移除;**最终在 S3 写 ChromaStore 时定**,本阶段只记 flag 不强删)。
**验收**:`test_smoke_imports.py` 全绿;`config.OCR_ENGINE` 已选定;冲突在此暴露(不拖到 S6)。

## 子阶段 S0.3 — 评估核心 TDD(尺子先于一切 · S0 重心)

**执行目标**:TDD 写出**不可变领域模型 + 纯函数评估核心 + 黄金 fixtures**,全绿、覆盖率达标。**先写测试再实现。**

**步骤**
1. `domain/models.py`:frozen dataclass/pydantic —— `FileRecord`、`Query`(+`structured`)、`CapabilityMatrix`、`FilterSpec`、`RetrievalResult`、`RetrievalTiming`/`IndexingTiming`、`MetricResult`(合同先行)。
2. `domain/match.py` + 测试:日期区间(epoch、半开 `[start,end)`)、子串(大小写不敏感 + NFC)、分词(jieba + 文件名分词器:拆驼峰/下划线/数字串、保留 `2307`)。**高风险 fixtures**:跨年、半开端点、NFC 中文子串、纯数字文件名。
3. `eval/metrics.py` + 测试:recall@k / precision@k / nDCG@k / MRR + `support`;`k_values` 批量;已知输入手算断言。
4. `eval/capability.py` + 测试:`supported_components`/`required_components`/`eligible`(子集判定);PRD §6.1 等价式 `✅ ⟺ required ⊆ supported` 逐格;视频 Q6 子集、媒体全真负例。
5. `eval/ground_truth.py`(**规则派生**部分)+ 测试:用迷你 manifest+matrix+structured 派生 `positive_ids`;**QA 映射 / 抽检手标路留 S2,仅留接口占位**。
6. `eval/attribution.py` + 测试:成员判定 TP/FP + `id→type` → 6×6;**归因四分支** fixture(有资格命中=TP、无资格命中=FP、视频地点分量、媒体误捞)。
7. `tests/fixtures/`:~10 文件迷你 manifest、六类型迷你矩阵、每类 1–2 条迷你 queries + 手算 GT;**mock `Retriever`** 脚本化 ranked。

**任务边界**:只写纯函数 + 模型 + fixtures + 测试;**不接 LlamaIndex/Chroma/真实数据/真实模型**;`ground_truth` 的 QA/手标、`attribution` 接真实检索结果留 S2/S4;`Reporter`/`EvalRunner`/索引/检索留后续阶段。
**验收**:`python -m pytest` 全绿;**覆盖率 ≥80%**(`capability`/`ground_truth`/`attribution` 更高);高风险 fixtures + 归因四分支均有断言;每模块 **<400 行**。
